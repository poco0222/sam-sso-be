# yr-system 最佳实践遗漏项修复设计

> **范围**: 仅修复本轮复查确认的方案 A 问题，不扩展到额外架构改造。

## 目标

在不扩大改动面的前提下，修复 `yr-system` 中已确认的 5 类问题：

1. `SysDeptMapper.xml` 新增部门时 `accounteUnit` 参数未绑定。
2. 批量删除用户时未清理 `sys_user_role`、`sys_user_post` 关联。
3. 批量删除角色时未清理 `sys_role_dept` 关联。
4. 批量删除字典类型缺少事务边界。
5. WebSocket 暴露了无效的 `async()` API，且仓库中没有真实调用。

## 设计原则

- 只做根因修复，不顺手扩展其它重构。
- 先写失败测试，再做最小实现，保持现有行为稳定。
- 优先修复数据一致性和实际运行错误，再处理最佳实践偏差。
- 如果某个问题在仓库内已经确认“无调用”，优先删除误导性接口而不是补复杂实现。

## 方案选择

### 方案 A（已批准）

- 修复 XML 参数绑定错误。
- 恢复批量删除用户/角色时的关联表清理。
- 为批量删除字典类型增加事务。
- 删除无效 WebSocket `async()` 接口与实现，并补回归测试。

### 不采用方案 B 的原因

- 不在本轮引入 `@EnableAsync` 和线程池装配链路，避免把修复范围扩大到 `yr-framework` / `yr-admin`。

### 不采用方案 C 的原因

- 会留下已确认的最佳实践偏差，不符合本轮“补遗漏项”的目标。

## 风险控制

- 所有关联清理都先通过单元测试锁定，再恢复服务层调用。
- 删除 `async()` 前先全文确认无调用方。
- 批量删除事务修复只加注解，不改变业务语义。
- 验证以 `mvn -pl yr-system test` 为准。
